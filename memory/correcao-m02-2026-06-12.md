## Problema
O M02 estava gerando justificativas genéricas ("Nota baseada em análise heurística") para todos os indicadores B1-B5 porque:
1. O SDK Anthropic estava apontando para `http://127.0.0.1:11434` (Ollama local) em vez da API real
2. O fallback para Ollama gerava JSON inválido
3. O código não carregava corretamente o `.env` com a chave de API

## Solução Implementada

### 1. Forçar uso da API Anthropic real
```python
api_key = os.environ.get("ANTHROPIC_API_KEY")
client = anthropic.Anthropic(api_key=api_key, base_url="https://api.anthropic.com")
```

### 2. Carregar .env com dotenv
```python
from dotenv import load_dotenv
load_dotenv(Path(__file__).parent.parent / ".env", override=True)
```

### 3. Prompt explícito para justificativas detalhadas
System prompt inclui instrução explícita:
> Para CADA indicador (B1-B5), você DEVE escrever uma justificativa ESPECÍFICA mencionando:
> - Trechos concretos do material (cite entre aspas)
> - Seções específicas (ex: "Seção 2, parágrafo 3")
> - Exemplos do que está bom ou ruim

### 4. Correção do logger
`logger.py` agora corrige estrutura inválida (_log.json corrompido como lista em vez de dicionário).
